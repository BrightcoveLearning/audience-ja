    オーディエンスの問題のトラブルシューティング

    このトピックでは、ギャラリーエクスペリエンスを通じて収集されたリードフォームデータをエクスポートする方法について説明します。

    詳細ログの有効化

    詳細ログを有効にすると、ブラウザーコンソールでオーディエンス関連情報を表示できます。これは、オーディエンスの問題をデバッグするときに役立ちます。 bcverbose=1詳細ログを有効にするには、Audience対応プレーヤーを使用してページの URL に追加するか、"verbose": trueまたはオプションで Audience プラグインを設定します。

    ログメッセージは、コンソールで表示できます。

    リードフォームに関する問題

    以下は、リードフォームで見た一般的な問題と質問です。

    フォームが送信された後も Marketo Munchkin 統合でリードフォームが表示され続けるのはなぜですか？

    Munchkin統合は、ユーザーが知られているかどうかを尋ねるために使用できるAPIを持っていません。既知のユーザーかどうかわからないので、常にリードフォームを表示します。Marketo REST 統合では API が提供されるため、既知のユーザーと不明なユーザーを特定できます。Marketa REST 統合に切り替えることが推奨され、この問題を回避できます。

    カスタムリードフォームは、送信時にリダイレクトするように設定されています。フォームを送信した後、1 秒または 2 秒間リダイレクトされるページが表示されるのはなぜですか？

    リードフォームを破棄する前に、リダイレクトされたページが完全に読み込まれるまで待たなければなりません。解決策の 1 つは、軽量（画像や JavaScript の多くがない）ページ、またはこの問題に対処するために特別にセットアップした（にあるhttps://map.brightcove.com/lead/complete）空白ページにリダイレクトすることです。

    確認HTMLでリードフォームを設定しました。提出後数秒間黒い画面が表示されるのはなぜですか？

    HTML のテキストの色を必ず変更してください。確認HTMLはおそらく表示されます。デフォルトのテキストの色は黒であるため、表示されません。

    HubSpot リードフォームが提出後に却下されないのはなぜですか？

    HubSpot のリードフォームがにリダイレクトするように設定されていることを確認してくださいhttp://map.brightcove.com/lead/complete。確認メッセージを表示する場合は、HubSpot ではなく Audience 内でこれを設定します。

    リードフォームが表示されないのはなぜですか？

    リードフォームが表示されない場合は、まず、リードフォームを表示するためのすべての基準が満たされていることを確認します。視聴している複数の動画に依存する基準は、ページの更新では機能しません。これらは、次のビデオが同じプレーヤーに自動的にキューに入れられるユースケースを対象としています。

    1. オーディエンスプラグインがプレーヤーに追加されていることを確認します。詳細については、「オーディエンス対応プレーヤーの作成」を参照してください。
    2. 統合が正しくセットアップされ、プラグインに送信されていることを確認します。
    3. リードフォームがそのプレイヤーに追加されていることを確認します。プラグインの設定に対する呼び出しに対する応答には、リードフォームがプレーヤーに追加されている場合、LeadForm プロパティの下にリードフォーム表示情報が含まれている必要があります。
    4. ビューアーがまだリードしていないことを確認します。リードフォームは、視聴者がマーケティングオートメーションプラットフォーム (MAP) のリードではないことをオーディエンスが特定した場合、またはリードフォームにまだ入力していない場合にのみ表示されます。プラグインは、ビューアがリードであるかどうかを確認するために、Audience サーバーに別の JSONP 呼び出しを行います。ビューアがリードの場合、サーバーはステータスが 1 で、ビューデータを MAP に同期するのに便利なその他の識別子で応答します。ビューアがリードでない場合、サーバーはステータスが 0 で応答します。
      • リードではないビューアに対するレスポンスの例: 
        /**/ typeof JSONpCallback_2935650618 == '関数' &&
 jsonpCallback_2935650618 ({"ステータス」: 0});
      • リードであるビューアに対するレスポンスの例: 
        /**/ typeof JSONpCallback_45936049288 == '関数' &&
 jsonpCallback_45936049288 ({"status」: 1, "eloqua」: {"ContactId」: "5"," InternalID」: "CTCPB000000000005"}});

    同期の問題

    以下は、私たちが見た同期の一般的な問題をいくつか挙げます。

    Marketo REST ビューイベントがエラーと同期に失敗したアクティビティタイプ ID '000000' が見つかりません

    これは、統合に関連付けられたカスタム Marketa アクティビティが削除された場合に発生します。Marketoの REST 統合が作成されると、アクティビティを取得し、その ID を保存し、その ID を使用してイベントを同期します。解決策（新しい Brightcove Video View アクティビティが作成されていることを前提として）は、統合を切断して再作成して、新しいアクティビティの ID を取得することです。

    同期していないイベントを表示

    統合が MAP で正しくセットアップされると、オーディエンス対応のプレーヤーからデータを表示すると、[ すべての最近のアクティビティ ] ページにデータが入力され、各 MAP の既知のリードに対応するイベントの表示が最終的にツールと同期されます。

    まず、[ 最近のすべてのアクティビティ ] ページにビューデータが表示されていることを確認します。

    ビューイベントが一覧表示されていても、マップ内にない場合は、API 経由で MAP にビューデータをインポートするための十分な権限がないユーザーとして、ユーザーが MAP でオーディエンスを承認しているか、統合が正しく構成されていない可能性があります。オーディエンスのホームページには、最近のビューイベントの同期操作のステータスが一覧表示されます。ビューイベントが MAP に同期されない理由を示すために、フィードバックがそこに表示されます。

    オーディエンスにビューイベントが表示されている場合は、[ ステータス ] 列が同期されているかどうかを確認します。これは、オーディエンスがデータを MAP に正常に配信できたことを示します。unknown_contact のステータスが表示された場合、これはユーザが知られていないため、MAP が受信できないことを意味します。ほとんどの MAP は、既知のユーザーのサードパーティデータのみを受け入れます。解決策は、マップで作成されたカスタムリードフォームを使用して、実際に既知のユーザーであることを保証することです。

    ビューイベントが一覧に表示されない場合は、オーディエンスがオーディエンス対応プレーヤーで設定されていない可能性があります。開発者ツールを有効にしたブラウザーを使用して、問題のあるビデオを含むページにアクセスし、特定の HTTP 呼び出しが Audience プラグインによって行われることを確認します。

    プレーヤーがロードされると、プラグインは Audience 設定エンドポイントを呼び出して、統合に基づいてプラグインの動作を決定するパラメーターを取得する必要があります。

    このコールがネットワークトラフィックから欠落している場合は、プレーヤーにインテグレーションが割り当てられていないことを意味します。これを行う方法については、「オーディエンス対応プレーヤーを作成する」を参照してください。

    この呼び出しは JSONPを使用して行われます。JSONPレスポンスが空のオブジェクトである場合、埋め込みコードが正しくないため、間違ったアカウント ID が指定されている可能性があります。設定要求にアカウントとプレーヤーの URL パラメータの両方を含めることができない場合、または指定されたアカウントを Audience-固有のデータベースから取得できなかった場合（ほとんどの場合、URL/埋め込みコードで間違ったアカウントが指定され、データベース)。

    /**/ typeof jsonpCallback_78754666587 == '関数' && jsonpCallback_78754666587 ({});

    オブジェクトに ID パラメーターがあり、統合パラメーターがない場合、ユーザーはマーケティングオートメーションツールとの統合を設定していないため、ビューイベントは記録されません。

    また、オーディエンス対応プレーヤーを公開するときに標準埋め込み (iframe) コードが使用された場合、データ同期の問題も生じます。プレーヤーを公開するときは、高度な埋め込み (ページ内) コードを使用します。標準埋め込み (iframe) 公開コードは、オーディエンスプラグインが iframe 内で実行されるため、機能しません。したがって、親ページの Cookie または  URL パラメータにアクセスできなくなります。

    接続の問題

    Audienceモジュールは、Eloqua統合を設定できないと言っています

    Eloqua の要件を確認してください。IPホワイトリストセキュリティ機能を使用している場合、指定されたIPアドレスをホワイトリストに登録してオーディエンス-Eloquaトラフィックを許可しない限り、オーディエンスはEloquaと通信できません。詳細については、Brightcove ブのサポートにお問い合わせください。

    オーディエンスは、Marketa RESTカスタムアクティビティがフィールドがないと言っていますが、Marketeで見ると、すべてのフィールドが存在することがわかります

    確認すべきことはいくつかあります：

    1. フィールド名は正確に一致する必要があります。「イベント 1」は「Event1」と同じではありません。
    2. アクティビティは承認済みで、ドラフト状態ではない必要があります。
    3. カスタムアクティビティを削除し、新しい Marketto REST 統合を設定せずに再作成した場合は、古いインテグレーションを切断し、新しい統合を作成する必要があります。カスタムアクティビティを検索し、統合ごとにその ID 値を保存します。IDを削除して変更した場合、新しいIDを自動的に取得することはできません。
    4. 新しいフィールドが最近アクティビティに追加された場合、Marketa のシステムに変更が有効になるまでに最大 24 時間かかることがあります。
