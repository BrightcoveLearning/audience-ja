    Audienceモジュールが視聴者を識別する方法

    このトピックでは、オーディエンスモジュールが視聴者を識別しようとするさまざまな方法について説明します。

    Audience モジュールは、さまざまなデータソースを使用して視聴者を識別しようとします。

    • オーディエンスクッキー
    • オラクル・エロクア GUID
    • マルケトマンチキンクッキー
    • Salesforce ID URL パラメータ
    • Salesforce ID HTML 属性
    • ハブスポット UTK クッキー
    • 電子メールパラメータ

    オーディエンスモジュールには、トラッキング拒否に関するユーザーのブラウザ設定が適用されます。

    注：カスタム統合を使用する場合は、ビューアを識別する責任があります。詳細については、「オーディエンスモジュールを使用したカスタム統合の作成」を参照してください。

    オーディエンスクッキー

     オーディエンス対応プレイヤーはサードパーティ Cookie を設定し、そのクッキーを使用して視聴データを同じユーザーに関連付けます。アカウントでオーディエンストラッキングが有効になっていて、プレーヤーがそれを使用している場合。ユーザーがオーディエンスリードフォームに入力した場合、オーディエンスはその閲覧者のプロファイルを作成し、すべての閲覧データがそのプロファイルに関連付けられます。

    オーディエンスが設定するクッキーの名前は _bc_uuid です。

    オラクル・エロクア GUID

    Eloqua インテグレーションを持つオーディエンス対応プレイヤーは、Eloqua の API を呼び出して訪問者 ID を取得します。これは多くの場合 Eloqua GUID と呼ばれます。この ID は、Audience モジュールの表示データに取り込まれ、表示データを Eloqua と同期するために使用されます。

    Eloqua は独自のクッキーセットを使用し、ブライトコーブプレーヤーが Eloqua インテグレーションで設定されている場合、オーディエンスは Eloqua クッキー内の GUID をキャプチャし、Eloqua サーバー側と通信し、ユーザーが誰なのかを調べます。

    マルケトマンチキンクッキー

    Marketo インテグレーションを持つオーディエンスが有効なプレイヤーは、ユーザーのブラウザーで作成された Munchkin クッキー (_mkto_trk) を参照します。この値は、Audience モジュールの表示データに取り込まれ、表示データを Marketto と同期するために使用されます。

    Salesforce ID URL パラメータ

    Salesforce 統合の場合、プラグインは bcsfid というパラメータの URL をチェックすることで、ビューアを識別できます。これにより、Salesforce ユーザは、URL にリードの Salesforce ID を指定することにより、トラッキングの目的で一意に生成されたランディングページ URL を使用してメールキャンペーンを送信できます。

    Salesforce ID HTML 属性

    Salesforce 統合の場合、プラグインが bcsfid URL パラメータをチェックしてビューアを識別できない場合、HTML <video> 埋め込みの要素の data-bc-sfid 属性を検索します。をパブリッシュし、その値をリードの Salesforce ID として使用し、設定されている場合、その値をリードの Salesforce ID として使用します。

    ハブスポット UTK クッキー

    HubSpot 統合の場合、プラグインは HubSpot の JavaScript トラッキングコードを注入し、ユーザーのブラウザーに hubspotutk クッキーをドロップします。この Cookie は、HubSpot 内で訪問者の身元を追跡し、ブライトコーブの閲覧データを連絡先に関連付けるために使用されます。プレーヤーのロード時にユーザーが既に hubspotutk クッキーを持っている場合、オーディエンスは HubSpot に API リクエストを行い、Cookie の値が既知の連絡先に対応しているかどうかを尋ねます。そうであれば、オーディエンスは閲覧データをそのユーザーに関連付けます。

    電子メールパラメータ

    オーディエンスモジュールは、電子メールアドレスを使用してユーザーが誰であるか伝えることもできます。オーディエンスは、 bcemail というタイトルの URL パラメータを探します。これは、トラッキングの目的で一意に生成されたランディングページ URL を持つメールキャンペーンで便利です。マーケティングプラットフォームから、または MailChimp などのサードパーティの電子メールサービスから、リストにバッチメールを送信する場合は、送信側プラットフォームの設定で E メールパラメーターを構成して、ユーザーがクリックしたリンクを動的に更新する必要があります。その後、オーディエンスはこの E メールパラメーターを使用してユーザーサーバー側を検索し、表示データを同期します。email パラメーターは、プレーヤーの埋め込みコードに埋め込むこともできます。

    URL の例

    http://go.brightcove.com/be-video-hero-cust?bcemail=msmith@brightcove.com

    オーディエンスプラグインが bcemail URL パラメータを使用して電子メールアドレスを見つからない場合は、HTML <video> 埋め込み公開コードの要素をチェックします。これは、ビデオ要素上のデータBC-電子メール属性を検索し、それが設定されている場合は電子メールアドレスとしてその値を使用します。

    埋め込みの例

    <ビデオ
id= "player」
data-account="0000000001"
データプレーヤー=「デフォルト」
data-video-id= "00000002"
データBC-メール= 」msmith@brightcove.com」
クラス=「ビデオJS」コントロール >
</ビデオ >

    追跡しない

    オーディエンス対応のプレイヤーは、トラッキング拒否 (DNT)  に関する視聴者のブラウザ設定に従うため、トラッキング拒否 (Do Nt Track) が有効になっている場合は、通常のクッキーを使用または設定しません。また、Video Cloud の顧客がすべての視聴者やユーザーごとに無効にしたい場合に、プレーヤープラグインをトラッキングしないように指示することもできます。これは、ページ内埋め込みコードを使用して、data-do-not-track の値を 1 に設定することによって行われます。

    <video-js data-video-id= "9240259371001"
data-account= "3491906377"
データ-トラックない= "1" 
data-player= "7dccc39d-0e2D-4b27-bea9-af44b2166148"
データ埋め込み=「デフォルト」クラス=「ビデオ-js」> < コントロール/video-js >
<スクリプト src=」https://players.brightcove.net/1486906377/7dccc39d-0e2d-4b27-bea9-af44b2166148_default/index.min.js "> < /script >

    Oracle Eloqua

    Eloqua は、要求を追跡せず、Eloqua GUID を返すように処理しない (基本的に無視する) と述べています。現在、オーディエンスはこれを特別な方法で処理していません。つまり、オーディエンスがDNTを見たときに Eloqua JavaScript API を使用しません。Eloqua は、EU プライバシー要件を満たす厳格なモードもサポートしています。Eloqua アカウントでこれが有効になっている場合、フォーム経由でオプトインしない限り、エンドユーザーは追跡されません。いずれの場合も削除された Cookie は、Oracle ドキュメントに記載されています。 Eloqua 追跡ページを参照すると、どのようなクッキーが削除されるのですか。

    Marketo

    Marketoでは、DNTをサポートしていますが、特定のアカウントでDNTをオン/オフするのはお客様に任されています。詳細については、Marketo ドキュメント「追跡しない」ブラウザサポート設定の編集を参照してください。

    HubSpot

    HubSpot はこの文書に従って DNT を尊重します。
