    Audienceモジュールの基礎

    このトピックでは、動画エンゲージメントを追跡するために使用されるオーディエンスモジュールの概要について説明します。
     

    オーディエンスモジュールは、オーディエンス対応の Brightcove Player 内で視聴された動画の動画エンゲージメントデータを追跡するために使用されます。閲覧データは Video Cloud に保存され、必要に応じて、Eloqua、Marketo、HubSpot、Salesforce などの一般的なマーケティング自動化プラットフォームと同期することができます。

    Audienceモジュールにアクセスするには、Video Cloudにログインし、ナビゲーションヘッダーの [Audience] をクリックします。オーディエンスのホームページには、動画が視聴された時間、視聴された動画、誰が視聴されたかについての洞察を提供する動画エンゲージメントデータが表示されます。これによって、視聴者のエンゲージメントとコンバージョンを改善できます。

    ホームページでは以下ができます。

    • ビデオのパフォーマンスを見る
    • リードのパフォーマンスの表示
    • オーディエンスモジュールのステータスの表示

    Audienceモジュールでは、以下が可能です。

    • データ接続のセットアップ
    • オーディエンスのプレーヤーを作成する
    • リードフォームの作成と管理
    • メールに公開
    • プレイヤーイベントを見る
    • ビューアープロファイルの表示
    • 同期アクティビティの表示
    • オーディエンス設定の構成
    • オーディエンスデータのエクスポート
    • ギャラリーのリードをエクスポート
    • ビューアデータを管理する

    動画パフォーマンスを表示する

    動画パフォーマンスセクションでは、Audience対応プレーヤーの内部で再生された動画について、動画パフォーマンスの概要を表示します。

    注：オーディエンス分析システムは一般的な Video Cloud 分析システムとは別のもので、2 つのトラックデータは異なります。したがって、2つを比較すると、いくつかの指標に違いが表示されます。たとえば、Video Cloud 分析システムは、動画が開始されるとすぐに視聴数をカウントしますが、オーディエンス分析システムでは 10 秒以内に視聴回数がカウントされません。したがって、14 人の視聴者が動画を開始しても、10 秒前に動画を停止した場合、Video Cloud アナリティクスでは 14 回の再生回数が表示され、オーディエンスモジュールには 8 が表示されます。

    まず、チャートの期間を選択します。以下のオプションが利用できます。

    • 7日
    • 30日間
    • 90日

    以下に基づいたグラフが表示されます。

    • [視聴回数の多い動画]:
    • 最も魅力的な動画
    • 視聴者の傾向

    視聴回数の多い動画

    [最も閲覧した動画] グラフには、選択した期間における Audience対応プレーヤーで最も視聴された動画が表示されます。

    エンゲージメントの高い動画

    [最も魅力的なビデオ] グラフには、選択した期間における、オーディエンス対応プレーヤーで最も魅力的な動画が表示されます。エンゲージメントとは、視聴された割合の平均です。動画エンゲージメントの計算方法の詳細については、トレーニングビデオの「動画エンゲージメントスコアの計算」をご覧ください。

    視聴者の傾向

    [視聴者の傾向] グラフには、選択した期間におけるビデオの視聴回数が表示されます。既知の視聴者と匿名の視聴者の両方の視聴が表示されます。グラフをポイントすると、実際のデータポイントが表示されます。

    注：Marketa Munchkin 接続を使用する場合、視聴者の傾向グラフには不明な視聴者のデータのみが表示されます。Munchkin 接続を使用する場合、ユーザーの詳細は収集されません。より詳細なユーザー情報については、Marketa REST 接続を使用する必要があります。詳細については、「 REST API を使用したビデオクラウドと Marketo の統合」を参照してください。

    リードのパフォーマンスの表示

    リードの情報を表示するには、[ リード ] タブをクリックします。[リード] タブは、アカウントにオーディエンスリードフォームが作成されている場合にのみ表示されます。リードパフォーマンスセクションでは、オーディエンス対応プレーヤーの内部で再生されたビデオに対して、高いレベルのリードパフォーマンスを提供します。まず、チャートの期間を選択します。以下のオプションが利用できます。

    • 7日
    • 30日間
    • 90日

    以下に基づいたグラフが表示されます。

    • オーディエンスファネル
    • 動画の変換トップ
    • リードチャート

    視聴者ファネル

    オーディエンスファネルには、選択した期間におけるオーディエンスリードフォームを使用して、オーディエンス対応プレーヤーの動画の指標が表示されます。

    • 動画のインプレッション -読み込まれた動画の数
    • 動画再生回数 -ストリーミングの開始時に録画された動画の再生回数（巻き戻しまたは再生は含まれません）。個々の視聴者の測定値ではありません。
    • 引合フォームビュー -引合フォームビューの合計数
    • コンバージョン -送信されたリードフォームの合計数

    人気コンバージョン動画

    コンバージョン率の高い動画チャートには、コンバージョン率が最も高い動画が表示されます。コンバージョン率は、送信されたリードフォームの数を動画のインプレッション数で割って算出されます。

    リードグラフ

    「引合チャート」には、選択した期間における発行済引合フォームの数が日別に表示されます。グラフをポイントすると、実際のデータポイントが表示されます。

    オーディエンスモジュールのステータスの表示

    [対象ユーザーのステータス] セクションには、対象者モジュールの現在のステータスが表示されます。

    データ接続の設定

    [ データ接続 ] オプションでは、オーディエンス統合を構成します。オーディエンスは、一般的なマーケティング自動化プラットフォーム（「MAP  統合」）と統合することも、閲覧データをローカルに保存してからAudience API を使用してエクスポートまたは抽出することもできます（「マップ以外」統合）。次の表に、サポートされている統合の概要を示します。

    MAP 統合 マップ以外の統合
    • オラクル・エロクア
    • マルケト (REST API)
    • マルケト (Munchkin)
    • セールスフォース
    • ハブスポット (REST API)
    • HubSpot (クライアントサイド API)
    • カスタム統合
    • シングルサインオン
    • オーディエンストラッキング

    オーディエンス対応プレーヤーを公開する場合は、高度な埋め込み (ページ内) 公開コードを使用します。標準埋め込み (iframe) 公開コードは、オーディエンスプラグインが iframe 内で実行されるため、機能しません。したがって、親ページの URL パラメータにアクセスできず、正しい Cookie を取得できない可能性があります。

    注：設定できる接続数は、Video Cloud サブスクリプションに基づいています。

    オーディエンスプレーヤーの作成

    [ オーディエンスプレーヤー ] オプションを使用すると、オーディエンス対応の Brightcove プレーヤーを作成できます。動画エンゲージメントデータは、視聴者が有効な動画プレーヤー内で動画コンテンツを表示したときにのみ記録されます。オーディエンス対応プレーヤーの作成については、「オーディエンス対応プレーヤーの作成」を参照してください。

    リードフォームを作成、管理する

    [ リードフォーム ] オプションを使用すると、データキャプチャ用のリードフォームを作成できます。リードフォームを作成した後、Audience対応プレーヤーと関連付けることができます。潜在顧客フォームの作成の詳細は、「 対象者リードフォームの作成」を参照してください。サードパーティのマーケティング自動化プラットフォーム内で作成されたリードフォームを作成し、使用することもできます。

    動画をメールに公開する

    [ メールに公開 ] オプションでは、プレーヤー、ビデオを選択し、HTML 埋め込みコードを生成してビデオを電子メールで送信できます。詳細については、「オーディエンスモジュールを使用したビデオの電子メールへの公開」を参照してください。

    注：Media モジュールには、ビデオを電子メールで公開する機能もあります。詳細については、「ビデオを電子メールにパブリッシュする」を参照してください。

    プレーヤーイベントを表示する

    [ 最近のすべてのアクティビティ ] オプションでは、過去 24 時間記録されたプレイヤービューイベントを表示できます。詳細については、「オーディエンスモジュールでのプレーヤーイベントの表示」を参照してください。

    ビデオアクティビティの表示

    [ ビデオ (マップ以外) ] オプションを使用すると、ビデオの表示アクティビティを表示できます。詳細については、「ビデオアクティビティの表示」を参照してください。

    ビューアプロファイルの表示

    [ ビューアープロファイル (マップ以外) ] オプションを使用すると、プレーヤーの表示イベントをビューア別に表示できます。このオプションは、SSO またはカスタムインテグレーション、またはオーディエンストラッキングを使用している場合に有効になります。詳細は、「ビューアプロファイル情報の表示」を参照してください。

    同期アクティビティの表示

    [ 同期アクティビティ ] オプションには、オーディエンスからマーケティングオートメーションプラットフォームに同期された最新の 10 個のジョブが表示されます。同期は 1 時間単位で実行されます。接続を選択し、[ 同期のテスト ] ボタンをクリックして、同期を実行します。これは、新しい統合をテストし、イベントデータがマーケティングオートメーションプラットフォームに送信されるかどうかを待たずに済むようにする場合に役立ちます。テスト同期は、5 分に 1 回、1 時間に 10 回まで実行できます。

    注：カスタム統合を使用する場合、同期ジョブは作成されません。同期ジョブは、マーケティング自動化プラットフォームとの統合を使用する場合にのみ作成されます。

    オーディエンス設定の構成

    [ 設定] オプションでは、アカウントの既定の設定を構成できます。詳細については、「オーディエンス設定の構成」を参照してください。

    オーディエンスデータのエクスポート

    [ オーディエンスデータのエクスポート ] オプションは、ビューイベントとリードフォームデータをエクスポートするために使用します。このオプションは、アカウント管理者に対してのみ表示されます。オーディエンスデータのエクスポートについては、「オーディエンスモジュールからのオーディエンスデータのエクスポート」を参照してください。

    ギャラリーリードのエクスポート

    ギャラリーエクスペリエンスからキャプチャされたリードフォームデータをエクスポートするには、[ギャラリーリードをエクスポート] オプションを使用します。このオプションは、アカウント管理者に対してのみ表示されます。ギャラリーリードデータのエクスポートの詳細は、「ギャラリーリードデータのエクスポート」を参照してください。

    ビューアデータの管理

    [ ビューアデータを管理 ] オプションを使用すると、ユーザを検索し、そのユーザを削除するか、そのユーザのデータをエクスポートすることができます。このオプションは、アカウント管理者に対してのみ表示されます。ビューアデータの管理の詳細は、「ビューアデータの管理」を参照してください。
