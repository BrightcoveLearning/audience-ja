    シングルサインオンとオーディエンスの統合

    このトピックでは、シングルサインオンで使用するために Audience モジュールを構成する方法について説明します。

    オーディエンスモジュールは、ビューアがシングルサインオン（SSO）を使用してシステムにログインしたときの視聴アクティビティを追跡する機能を提供します。このシナリオでは、表示データは、マーケティング自動化プラットフォームに送信されません。オーディエンス SSO 設定では、SAML 属性がオーディエンスに渡され、ビューアを識別します。これの一般的な使用例の1つは、内部トレーニングビデオです。従業員は、ビデオを視聴するためにログインする必要がある場合があります。その後、視聴アクティビティを追跡し、オーディエンスモジュール内で表示できます。オーディエンスをSSOで使用できるようにするには、以下の手順が必要です。

    1. シングルサインオンソフトウェアの構成
    2. SSO 属性にアクセスするようにアプリケーションを設定する
    3. オーディエンス SSO 接続のセットアップ
    4. 統合の構成
    5. オーディエンス対応プレーヤーを作成する
    6. ビデオデータの表示

    SSO ソフトウェアの構成

    SSO ソフトウェアをインストールおよび構成する手順は、このトピックの対象外です。SSO ソフトウェアは、Video Cloud ビデオを見る前に、ユーザーが自身を認証する必要があるように設定する必要があります。SSO ソフトウェアは、外部アプリケーションからアクセスできるように、一部のユーザー属性を公開するように構成する必要があります。たとえば、この Okta SSO 設定では、FirstName、LastNameおよび 3 Emailつのアトリビュートが公開されます。

    このトピックでは、Email属性を使用して、オーディエンスモジュールの閲覧者を識別します。

    注：SSO ソフトウェアは複数の属性を公開するように構成できますが、オーディエンスはビューアを追跡するために 1 つの属性のみを使用します。

    SSO 属性にアクセスするためのアプリケーションの設定

    Video Cloud 動画の表示に使用するアプリケーションまたは Web サイトは、SSO 属性を JavaScript を使用してオーディエンスが利用できるように設定する必要があります。この設定は、使用されているアプリケーションまたは Web サイトによって異なります。次の例では、SSO アクセス制御プロファイルが Gallery に作成されています。構成の一部として、Email SSO 属性が公開され、ギャラリーサイト内で使用できるようになります。これは、オーディエンスモジュールの閲覧者を識別するために使用される値です。

    注：ギャラリーでのアクセス制御プロファイルの構成の詳細は、「アクセス制御プロファイルの構成」を参照してください。
    注：複数の属性を公開するようにギャラリーを設定できますが、オーディエンスはビューアの追跡に 1 つの属性しか使用しません。

    接続を設定する

    以下の手順に従って、Video Cloud と SSO プロバイダー間の接続を設定します。

    ビデオクラウドと SSO 間の接続の確立

    1. Audience モジュールを開き、左側のナビゲーションで [ データ接続 ] をクリックします。
    2. [新規コレクションの追加]をクリックします。
    3. 統合として SSO を選択します。
      注：利用可能な統合は、Video Cloud サブスクリプションに基づいています。
    4. （オプション）接続名を入力します。
    5. [接続]をクリックします。

    これで SSO 接続がセットアップされ、使用できる状態になります。

    統合を設定する

    SSO 接続を確立したら、統合を構成できます。

    1. [ SSO プロパティ ] emptyセクションのリンクをクリックします。ページ内で検索する JavaScript 値の名前を入力します。この例では、Gallery はEmail SSO 属性を公開するように構成されており、ここで使われている属性になります。

      このトピックでは、オーディエンス SSO 統合を使用して視聴アクティビティを追跡します。[保存]をクリックします。

      注：SSO プロパティを空のままにして、追加オプションの 1 つを使用してビューアを識別することもできます。
    2. デフォルトでは、統合ステータスは有効になります。統合ステータスに応じて、Video Cloud動画エンゲージメントデータが保存されるかどうかが決まります。
      • 有効 -データが収集されます
      • 無効 -データは収集されません（統合は無効です）
    3. データ保持ポリシーを選択します。データ保持ポリシーによって、視聴データおよび（Audienceリードフォームを使用している場合）リードフォームデータがご自身のVideo Cloudアカウントに保持される期間が決まります。保持設定は、プライバシーポリシーに従って調整する必要があります。

    統合を削除するには、[ 切断 ] をクリックします。ページの下部には、すべてのオーディエンスの有効なプレーヤー、それらが関連付けられている接続、および関連するリードフォームが表示されます。

    Audienceモジュールは、マーケティングプラットフォームへの複数の接続をサポートしています。追加の統合を設定するには、[新規接続の追加]をクリックします。

    ビューアーを識別するための追加オプション

    統合の構成時に SSO プロパティの値を指定しない場合は、閲覧者を識別する責任があります。オーディエンスモジュールには、次の情報が必要です。

    説明 URLパラメーター 埋込みパラメーター 値
    GUIDは何か（視聴イベントトラッキングに必要か） bcssoguid data-bc-sso-guid string

    このパラメーターは、URL パラメーターまたはプレーヤーの埋め込みコードを介して渡すことができます。視聴者のトラッキングが発生するには、このパラメーターは、動画を表示しているページの URL パラメーターとして渡す必要があります。

    注：プレーヤー/プラグインが URL パラメータを読み取るためには、Advanced (ページ内) 埋め込みコードを使用する必要があります。標準 (iframe) 埋め込みコードは、プラグインが iframe 内で実行されるため、パラメータを持つ親ページの URL にアクセスできないため、動作しません。

    ページで利用可能なJavaScript値を使って、GUIDをAudienceプレーヤーに渡すことも可能です。プレーヤー設定を使用して、JavaScript変数の名前をAudienceに伝えることができます。すると、Audienceはその変数を読み、カスタムGUIDとして使用しようとします。これを行うには、Audienceプレーヤーを編集し、Audienceプラグインのプラグインオプションを設定してください。

    上の例では、ページに samlAttributes というJavaScriptオブジェクトが存在し、それには nameID というプロパティがあることをプレーヤー設定はAudienceに伝えています。Audienceはそれを読み、カスタムGUIDにその値を使用しようとします。

    Audience対応プレーヤーを作成する

    動画エンゲージメントデータは、オーディエンス対応の動画プレーヤーを使用している場合にのみ追跡されます。Audience対応Brightcove Playerの作成については、「Audience対応プレーヤーを作成する」を参照してください。

    ビデオデータの表示

    SSO 統合を使用する場合、ビデオデータを表示するためのオプションがいくつかあります。

    • ビューアアクティビティ（イベント） -オーディエンスモジュールの一部として提供され、このオプションを使用して、オーディエンスモジュール内のビデオデータを表示できます。詳細については、「オーディエンスモジュールでのプレーヤーイベントの表示」を参照してください。
    • ビューアプロファイル -Audience モジュールの一部として提供され、このオプションを使用して、オーディエンスモジュール内のユーザープロファイル値に基づいてビデオデータを表示できます。詳細は、「ビューアプロファイル情報の表示」を参照してください。
    • データのエクスポート -オーディエンスモジュールの一部として提供され、このオプションを使用してビデオビューとリードデータをエクスポートできます。詳しくは、オーディエンスモジュールからのオーディエンスデータのエクスポートを参照してください。
    • オーディエンス API -オーディエンス API を使用して、ビデオビューとリードデータをプログラムで取得できます。詳細については、「概要」を参照してください。オーディエンス API 。
